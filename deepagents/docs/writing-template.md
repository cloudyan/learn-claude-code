# Chapter Writing Template

每一章建议统一采用下面的结构：

## 这一章解决什么问题

先说清楚上一章方案为什么不够。

## 为什么需要这个新机制

聚焦失败模式，不要一上来讲 API。

## 核心概念

只讲本章必须理解的 1 到 3 个概念。

## 最小示例

提供一个最小但可运行的 TypeScript 示例。

## 真实 Coding Task 示例

把本章能力放进一个更像真实工作的任务里。

## 设计取舍

说明为什么这样设计，不这样设计会怎样。

## 本章结论

用 3 到 5 句话总结读者应该记住什么。

## 下一章预告

解释为什么还需要继续升级。
