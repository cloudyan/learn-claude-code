# 第6章：总结与下一步

**7 个核心思想，和下一步建议。**

---

## 核心思想回顾（7 点）

这 7 点是贯穿整个教程的核心：

### 1. 模型即代理

> **模型是决策者，代码只提供工具并运行循环。**

- 不要在代码里写复杂逻辑
- 让模型决定用哪些工具、什么顺序、何时停止
- 代码就是 `while (true)` + 工具执行

### 2. 四个工具就够了

- `bash` - 运行命令
- `read_file` - 读取内容
- `write_file` - 写入文件
- `edit_file` - 精确修改

90% 的场景不需要第 5 个工具。

### 3. 先规划，再执行（Todo）

> **显式的任务列表，既是约束又是脚手架。**

- 计划可见，可追踪
- 软约束（Reminder）比硬编码好
- 给模型一些结构，但不要过度

### 4. 分而治之（Task）

> **Task 不是并行，是隔离。**

- 子代理有干净的上下文
- 可以给不同子代理不同的工具、不同的提示
- 不需要"多代理协作"，只需要"任务分解"

### 5. 知识外化（Skills）

> **不要在 prompt 里堆金子，按需加载。**

- 工具 vs 技能：能做什么 vs 知道什么
- SKILL.md：YAML frontmatter + Markdown
- 技能太多？加个"技能索引"技能

### 6. 终极简化（v0）

> **如果只能有一个工具，那就是 bash。**

- Unix 哲学：一个工具只做一件事
- 100 行（甚至更少）代码，就能做事
- 简单是复杂的基础

### 7. 可理解、可预测、可干预

- 不要让代理"胡作非为"
- 每步决策可见，输出可追踪
- 人可以随时介入

## 下一步建议

### 1. 先用起来

选一个日常任务，让代理帮你做：
- 重构一段代码
- 写点文档
- 做个小工具

从简单开始，慢慢加复杂度。

### 2. 观察，再调整

不要一开始就加 20 个工具。
- 先跑 v1
- 看哪里痛
- 再加 Todo / Task / Skills

### 3. 写你自己的 Skills

把你的领域知识写成 SKILL.md：
- 你们团队的代码风格
- 你们项目的架构约定
- 你们常用的设计模式

按需加载，而不是堆在 prompt 里。

### 4. 看看别人怎么做

这个领域还在快速发展。看看：
- OpenAI 的 Assistants API
- Anthropic 的 Claude Computer Use
- 各种开源框架（LangChain 等）

但记住：**原理都是一样的。**

## 最后一句

你已经拥有了构建任何 Agent 的核心工具。

- 不要被框架吓住
- 不要被"最佳实践"绑住
- 从简单开始，边做边学

**现在，去做你自己的 Agent 吧。**

## 配套代码

TypeScript 版本在 `typescript/` 目录：
```bash
cd agent-1/typescript
npm install

npm run v0   # 终极简化版
npm run v1   # 基础版（推荐入门）
npm run v2   # Todo 版
npm run v3   # 子代理版

```

Python 版本作为备份在 `python/` 目录。
