## 这不是什么，这是什么

### 这不是：
- ❌ 调包手册（"用这个框架，3 行代码搞定"）
- ❌ API 文档翻译
- ❌ 概念堆砌（"什么是 Agent？什么是 Tool？"）

### 这是：
- ✅ 从 0 到 1 手写 mini-claude-code 指南
- ✅ 讲清楚"为什么"
- ✅ 让你真正理解核心逻辑

---

## 你需要什么

- 基础 JavaScript 或 TypeScript（知道函数、循环、对象）
- 一个 API Key（OpenAI 或国内模型都可以）
- 愿意动手敲代码

就这么多。

不需要懂机器学习，不需要复杂框架，也不需要 GPU。

---

## 我们要写什么

从 **v1** 开始（最实用的版本），然后逐步升级到 v2、v3、v4，最后用 **v0** 来收尾（最极简的版本）。

| 版本 | 主题 | 核心洞察 |
|------|------|----------|
| v1 | 模型即代理 | 模型是 80%，代码只是循环 |
| v2 | 结构化规划 | Todo 让计划可见 |
| v3 | 分而治之 | 子代理隔离上下文 |
| v4 | 知识外化 | Skills 注入专业知识 |
| v0 | 终极简化 | Bash 就是一切 |

## 先剧透一点：核心模式

所有 Agent，不管多复杂，本质都是这个循环：

```typescript
// 伪代码
while (true) {
  const response = await model(messages, tools);
  if (response.stopReason !== "tool_use") {
    break;
  }
  const results = await executeTools(response.toolCalls);
  messages.push(...results);
}
```

就这么简单。

其他一切——待办、子代理、权限、Skills——都是围绕这个循环的精化。

## 配套代码

